updateChildren 。<br>patchVnode 有两个参数，分别是老节点 oldVnode, 新节点 vnode 。主要分五种情况：</p> <p>if (oldVnode &#x3D;&#x3D;&#x3D; vnode)，他们的引用一致，可以认为没有变化。<br>if(oldVnode.text !&#x3D;&#x3D; null &amp;&amp; vnode.text !&#x3D;&#x3D; null &amp;&amp; oldVnode.text !&#x3D;&#x3D; vnode.text)，文本节点的比较，需要修改，则会调用Node.textContent &#x3D; vnode.text。<br>if( oldCh &amp;&amp; ch &amp;&amp; oldCh !&#x3D;&#x3D; ch ), 两个节点都有子节点，而且它们不一样，这样我们会调用 updateChildren 函数比较子节点，这是diff的核心，后边会讲到。<br>if (ch)，只有新的节点有子节点，调用createEle(vnode)，vnode.el已经引用了老的dom节点，createEle函数会在老dom节点上添加子节点。<br>if (oldCh)，新节点没有子节点，老节点有子节点，直接删除老节点。</p>
